# Contributing to RAG PDF Search

Thank you for your interest in contributing to this project!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/rag-pdf-search.git`
3. Create a virtual environment: `python3.13 -m venv .venv`
4. Activate it: `source .venv/bin/activate`
5. Install dependencies: `pip install -r requirements.txt`

## Development Workflow

1. Create a new branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test your changes in the Jupyter notebooks
4. Commit with clear messages: `git commit -m "Add: feature description"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable names
- Add docstrings to classes and methods
- Keep functions focused and small

## Testing

Before submitting a PR:
- Test your changes with sample PDFs
- Verify embeddings are generated correctly
- Check that retrieval returns relevant results

## Reporting Issues

- Use the GitHub Issues tab
- Provide clear description and reproduction steps
- Include system information (Python version, OS)
- Share error messages and logs

## Suggestions

Have ideas for improvements? Open an issue with the "enhancement" label!
